    how to add to GRUB

    edit /etc/default/grub and find this line:

    /etc/default/grub
    GRUB_INIT_TUNE="480 440 1"

    replace with your tune. remove the # if present. save, then run:

    terminal
    $ sudo update-grub
    ⚠ WARNING GRUB plays tunes before showing the boot menu. playback of tunes will delay booting indefinitely until completed.

    syntax reference

    tempo freq duration [freq duration ...]

    each variable is only valid as a whole number. tempo can only be defined once, at the start of the GRUB_INIT_TUNE value. freq value must be followed with a duration value.

    tempo

    base timing in ticks per minute. not to be confused with BPM.

    more info

    a larger tempo value results in higher resolution for more precise duration values. increasing this value in an existing GRUB_INIT_TUNE value will result in faster playback, and slower if decreased.

    for converting from MIDI, tempo can be calculated by multiplying the beats per minute BPM by the number of ticks per beat or PPQ (parts per quarter note), and then rounding this value to the nearest whole number:

    grubTempo = ROUND(BPM × PPQ)

    ... where PPQ is the file's resolution (e.g., 480, 960). MIDI files store PPQ value in their header. you may find more information about how this web tool gathers and uses this information here.

    freq

    frequency of a note in Hz. one of the two required values to create one note.

    more info

    Hz for most notes rarely correlate to an integer, so it is best practice to round up or down to the nearest whole number (close enough for most human ears).

    middle C, referred to as C4 in scientific pitch notation and note number 60 in MIDI standard, is generally recognized as approximately 261.626 Hz. so for our purposes:

    262 Hz = C4

    0 = silence/rest

    refer to this image for a helpful reference table for converting standard notation, MIDI note numbers, and Hz respectively.

    duration

    duration of a note in ticks, relative to tempo (ticks per minute). the final value required to create a valid note.

    more info

    0 = nothing, skipped/ignored by GRUB.

    60 262 1 ... will result in the C4 note lasting for 1 second (60 seconds = 1 minute).

    as an example, let's say you wanted to write the pentatonic scale from C4 in 4/4 time.

    C4, D4, E4, G4, and A4.

    step 1: choose tempo and BPM

    we'll use 480 as our tempo and 120 BPM. why tempo = 480? it provides enough resolution for common note values to land on integers. fractional values are invalid in GRUB.

    sixteenth note = 1 tick

    eighth note = 2 ticks

    quarter note = 4 ticks

    whole note = 16 ticks

    remember, tempo and duration operate in ticks, not beats. BPM does not exist in GRUB or define such a value. we're working with a bootloader operating hardware on a low level, so the luxuries provided by MIDI through DAWs and sequencers do not apply.

    duration = 0 does nothing (gets skipped). decimals aren't supported. a lower tempo like 60 would make quarter notes impossible. that would require duration = 0.5, which is invalid.

    step 2: convert BPM to seconds

    at 120 BPM, we can calculate note durations in seconds:

    120 quarter notes per minute

    60 seconds ÷ 120 beats = 0.5 seconds per quarter note

    4 quarter notes = 1 whole note = 2 seconds

    step 3: convert seconds to ticks

    the relationship: seconds per note = (duration in ticks) ÷ (tempo in ticks/min) × 60

    with tempo = 480:

    1 minute = 480 ticks

    1 second = 8 ticks (480 ÷ 60)

    quarter note (0.5 sec) = 4 ticks (8 × 0.5)

    step 4: convert notes to Hz and ticks

    now translate to GRUB format. get your tempo, 480, and figure out the frequency for each note you desire (rounded to the nearest integer) and its duration in ticks.

    C4 (261.63 Hz, quarter note): 262 4

    D4 (293.67 Hz, quarter note): 294 4

    E4 (329.63 Hz, quarter note): 330 4

    G4 (392 Hz, quarter note): 392 4

    A4 (440 Hz, quarter note): 440 4

    final output:

    480 262 4 294 4 330 4 392 4 440 4
