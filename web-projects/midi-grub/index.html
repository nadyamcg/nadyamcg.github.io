    how to add to GRUB

    edit /etc/default/grub and find this line:

    /etc/default/grub
    GRUB_INIT_TUNE="480 440 1"

    replace with your tune. remove the # if present. save, then run:

    terminal
    $ sudo update-grub
    ⚠ WARNING GRUB plays tunes before showing the boot menu. playback of tunes will delay booting indefinitely until completed.

    syntax reference

    tempo freq duration [freq duration ...]

    each variable is only valid as a whole number. tempo can only be defined once at the start of the GRUB_INIT_TUNE string. freq values must be followed by a duration value.

    tempo

    base timing in ticks per minute. not to be confused with BPM.

    more info

    a larger tempo value results in higher resolution for more precise duration values. increasing this value in an existing GRUB_INIT_TUNE value will result in faster playback, and slower if decreased.

    for converting from MIDI, tempo can be calculated by multiplying the beats per minute BPM by the number of ticks per beat or PPQ (parts per quarter note), and then rounding this value to the nearest whole number:

    grubTempo = round(BPM × PPQ)

    round(x) =
    ⌊x⌋ if x − ⌊x⌋ < 0.5
    ⌊x⌋ + 1 if x − ⌊x⌋ ≥ 0.5

    ... where PPQ is the file's resolution (e.g., 480, 960). MIDI files store PPQ value in their header. you may find more information about how this web tool gathers and uses this information here.

    freq

    frequency of a note in Hz

    more info

    freq is one of the two required values in order to create a valid note in GRUB.

    notes are distinguished by their frequency measured in Hz. Hz values for most notes rarely correlate to an integer, this means we must round them the nearest whole number. as the quantization is ±1 Hz, this is close enough for most human ears.

    middle C, referred to as C4 in scientific pitch notation and note number 60 in MIDI standard, is generally recognized as approximately 261.626 Hz. the standard formula for deriving Hz from a MIDI note number is:

    freq = round(440 × 2(n − 69) ÷ 12)

    ... where n is the MIDI note number and 440 is the reference pitch A4 = 440 Hz. so, for C4 (MIDI note 60):

    freq = round(440 × 2(60 − 69) ÷ 12)

    freq = round(261.626)

    freq = 262

    262 Hz = C4

    0 = rest (silence)

    refer to this image for a helpful reference table for converting between standard notation, MIDI note numbers, and Hz values respectively.

    duration

    duration of a note in ticks, relative to tempo.

    more info

    duration is the other required value for each note. it tells GRUB how long to hold the frequency before moving to the next note. it's measured in ticks, not seconds or milliseconds.

    the actual length of time a tick represents depends entirely on the tempo value:

    seconds per tick = 60 ÷ tempo

    note length = duration × 60 ÷ tempo

    for example, 60 262 1 plays C4 for 1 second, because tempo = 60 means 60 ticks per minute, so each tick = 1 second.

    note length = 1 × 60 ÷ 60 = 1 second

    higher tempo values give you finer duration control. at tempo = 480, each tick is only 0.125 seconds (125ms), allowing precise note lengths without needing fractional values.

    0 = nothing, skipped/ignored by GRUB.

    example manual workflow

    let's say you wanted to write the pentatonic scale from C4 in 4/4 time, which is:

    C4 D4 E4 G4 A4

    step 1: choose tempo and BPM

    we'll use 480 as our tempo and 120 BPM. why tempo = 480? it provides enough resolution for common note values to land on integers. fractional values are invalid in GRUB.

    remember, tempo and duration operate in ticks, not beats. BPM does not exist in GRUB or define such a value. we're working with a bootloader operating hardware on a low level, so the luxuries provided by MIDI through DAWs and sequencers do not apply.

    a lower tempo like 60 would make quarter notes impossible. that would require duration = 0.5, which is invalid.

    step 2: convert BPM to seconds

    at 120 BPM, we can calculate note durations in seconds:

    120 quarter notes (or beats) per minute

    60 seconds ÷ 120 beats = 0.5 seconds per quarter note

    4 quarter notes = 1 whole note = 2 seconds

    step 3: convert seconds to ticks

    the relationship:

    seconds per note = duration ÷ tempo × 60

    with tempo = 480:

    1 minute = 480 ticks

    1 second = 8 ticks (480 ÷ 60)

    quarter note (0.5 sec) = 4 ticks (8 × 0.5)

    we've identified that quarter notes in our context should equal to 4. applying the same math to other note types:

    sixteenth note (0.125 sec) = 1 tick (8 × 0.125)

    eighth note (0.25 sec) = 2 ticks (8 × 0.25)

    quarter note (0.5 sec) = 4 ticks (8 × 0.5)

    half note (1 sec) = 8 ticks (8 × 1)

    whole note (2 sec) = 16 ticks (8 × 2)

    step 4: convert notes to Hz and ticks

    now translate to GRUB format. each note needs a freq and a duration. we already know our duration is 4 ticks (quarter note), so for each note we just need to round its Hz value to the nearest integer:

    C4 (261.626 Hz) -> round(261.626) = 262 -> 262 4

    D4 (293.665 Hz) -> round(293.665) = 294 -> 294 4

    E4 (329.628 Hz) -> round(329.628) = 330 -> 330 4

    G4 (391.995 Hz) -> round(391.995) = 392 -> 392 4

    A4 (440 Hz) -> round(440) = 440 -> 440 4

    final output

    480 262 4 294 4 330 4 392 4 440 4
